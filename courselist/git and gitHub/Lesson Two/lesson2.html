Setting Up Git and Your First Repository

Lesson 2: Core Git Commands and Workflow

  • Understanding the Git Workflow:
    Git organizes changes to your files in three stages:

    • Working Directory: This is the folder where you edit and create files. Changes made here are not yet tracked by Git.
    • Staging Area: When you decide which files or changes to track, you move them to the staging area. Think of this as "preparing" changes for saving.
    • Repository: This is the place where your changes are saved permanently. Once saved, you can always return to a previous version if needed.

  • Core Git Commands:
    Here are the main Git commands to work with the workflow:

    1. git init: Initializes (sets up) a new Git repository in the current folder.
      • Example:
      • git init
      • After running this command, Git starts tracking changes in your project folder.
    2. git status: Shows the current state of your files.
      • Are there changes that are not yet staged?
      • Are there staged changes waiting to be committed?
      • Example:
      • git status
    3. git add <file>: Moves a file or changes to the staging area. Use this to prepare specific files for a commit.
      • Example:
      • git add file.txt
      • This command stages file.txt for committing.
    4. git commit -m "message": Saves changes from the staging area to the repository. Always include a meaningful message in quotes to describe the changes.
      • Example:
      • git commit -m "Initial commit"
    5. git log: Displays a history of commits in your repository. It shows details like the commit message, date, and author.
      • Example:
      • git log

  • Practical Exercise:
    Let’s put these commands into action step by step:

    1. Create a Folder and Initialize a Git Repository

    1. Create a new folder for your project:
      • mkdir my-git-project
      • cd my-git-project
    2. Initialize Git in this folder:
      • git init
    3. Now, your folder is ready to track changes with Git.

    2. Add and Commit a File

    1. Create a file named file.txt with some content:
      • echo "Hello, Git!" > file.txt
    2. Check the status of your repository:
      • git status
      • You’ll see file.txt as an untracked file.
    3. Add the file to the staging area:
      • git add file.txt
    4. Commit the file to the repository with a message:
      • git commit -m "Initial commit"
      • This saves the current version of file.txt to the repository.

    3. Check Your Commit History

    1. View the history of commits:
      • git log
      • This will show the commit ID, author, date, and the commit message ("Initial commit").

  • Summary:

    • Git Workflow: Working Directory → Staging Area → Repository
    • Commands to Remember:
      • git init: Set up a Git repository.
      • git status: Check the current state of the repository.
      • git add <file>: Stage changes.
      • git commit -m "message": Save changes permanently.
      • git log: View your commit history.
    • Practical Skills:
      • Create and initialize a repository.
      • Add, commit, and track changes.
      • Review your commit history.
