Branching and Merging in Git

Lesson 3: Branching and Merging

  • What is Branching?
    Branching in Git allows you to create a separate line of development in your project. Think of it as a way to experiment with changes without affecting the main codebase.

  • Key Commands for Branching:

    1. git branch <branch-name>: Create a new branch.
    2. git checkout <branch-name>: Switch to an existing branch.
    3. git checkout -b <branch-name>: Create and switch to a new branch in one step.

  • What is Merging?
    Merging allows you to integrate changes from one branch into another. Itâ€™s typically used to incorporate features or fixes into the main branch.

  • Key Commands for Merging:

    1. git merge <branch-name>: Merge the specified branch into your current branch.
    2. git log --graph: View a visual representation of branches and merges.

  • Practical Exercise:

    1. Create and Switch to a Branch

    1. Create a branch named feature-branch:
      • git branch feature-branch
    2. Switch to the new branch:
      • git checkout feature-branch

    2. Make Changes and Commit

    1. Edit or create files in your project.
    2. Stage and commit the changes:
      • git add .
      • git commit -m "Added new feature"

    3. Merge Changes into Main Branch

    1. Switch back to the main branch:
      • git checkout main
    2. Merge the changes from feature-branch:
      • git merge feature-branch
